# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✅ Yes    |

## Reporting Security Issues

If you believe you have found a security vulnerability in darkTunes Charts, please report it responsibly via coordinated disclosure.

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please open a [GitHub Security Advisory](https://github.com/Neuroklast/darktunes-charts/security/advisories/new) in this repository, or contact the project maintainer directly.

Please include as much of the following information as you can to help us understand and resolve the issue quickly:

- The type of issue (e.g., XSS, injection, authentication bypass)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## Scope

This policy covers the darkTunes Charts application hosted at this repository, including:
- Frontend React application (`src/`)
- Vercel Serverless API functions (`api/`)
- Vercel deployment configuration (`vercel.json`)

## Response Timeline

We aim to acknowledge reported vulnerabilities within **72 hours** and provide a fix or mitigation plan within **14 days** for critical issues.
